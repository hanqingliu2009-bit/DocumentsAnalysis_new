### 2026-04-23

#### 1. Documents 列表增加「页数」列；DOCX 补充 `metadata.page_count`

**原因：** PDF 处理结果里已有 `page_count`，但前端未展示；DOCX 未写入页数，无法在列表与 Word 页数对照，难以判断「版式页数是否被识别」。

- `backend/tests/core/test_processor.py`：为 `_read_docx_page_count` 增加单元测试。

**验证：** 待你本地确认（PDF/DOCX 各传一份，列表页数与 Word/PDF 页数一致；无 `<Pages>` 的 docx 仍为 —；历史已入库 DOCX 需重传后才有页数）。

### 2026-04-22

#### 1. 无检索命中时：通用 LLM 的 system 提示带上索引统计（避免「你没上传」式误导）

**原因：** Chat 在向量检索**零命中**时仍走 `_generate_answer_without_context`。旧 system 一味强调「去 Documents 上传、处理完成后再问」，模型容易把「本轮没匹配到片段」说成「没有可查询文档 / 仍未上传处理」，与 Dashboard 已有 chunk、Documents 已为 **completed** 的事实矛盾。

**改动内容：**
- `backend/core/rag.py`：新增 `_system_prompt_for_no_retrieval_match()`，读取 `vector_store.get_chunk_count()` 与 `document_store.count_by_status()` 中 **completed** 数量；**片段数为 0** 与 **大于 0** 分支说明不同；索引非空时明确要求模型**不要**断言用户未完成上传，并提示语义匹配、**SIMILARITY_THRESHOLD**、可看 Documents / Dashboard。
- `backend/api/query.py`：`RAGPipeline(..., document_store=main.document_store)`。
- `backend/tests/core/test_rag_no_retrieval.py`：为 mock 向量库补充 `get_chunk_count`；新增 `test_no_retrieval_system_prompt_when_index_non_empty`。

**验证：** 已验证（用户确认记入本日志）。

---

#### 2. 上传文档的 `document_id` 与向量库 chunk 一致（删除能清掉向量）

**原因：** 上传接口先创建并保存 `Document`（id=A），`DocumentProcessor.process_document` 内部又 `new` 了另一个 `Document`（id=B），分块写入 Chroma 时 `metadata.document_id` 为 **B**，而列表与删除接口使用的是 **A**。结果是：`DELETE` 无法按 A 删除向量、Dashboard **chunks 总数不随删文档减少**、已删文档仍可能被语义检索命中。

**改动内容：**
- `backend/core/processor.py`：`process_document(..., document=None)`，若传入 API 已创建的 `Document`，在同一实例上更新元数据与状态，且 `_create_chunks(..., document.id)` 使用 **A**。
- `backend/api/documents.py`：上传时传入 `document=document`；处理未 `COMPLETED` 时返回 500；`except` 中先 `raise HTTPException` 再捕获其它异常。
- `backend/tests/core/test_processor.py`：新增 `test_process_document_reuses_api_document_id`。
- `backend/tests/api/test_documents.py`：上传相关 mock 改为 `side_effect`，对传入的 `document` 置 `COMPLETED` 并返回同一引用。

**验证：** 已验证（删除文档后 chunk 数减少、不再检索到已删内容；历史脏数据需自行清空 `vector_db` 后重传）。

#### 3. Chat 对话在离开页面后仍保留（浏览器本地）

**原因：** 聊天记录仅存于 `ChatInterface` 的 `useState`，路由切到 Documents / Dashboard 后组件卸载，**对话丢失**。

**改动内容：**
- `frontend/src/pages/ChatInterface.tsx`：使用 `localStorage` 键 `documents-analysis-chat-messages-v1`；初次挂载从本地恢复；`messages` 变化时写入；确认清空对话时移除该项。

**验证：** 已验证（离开 Chat 再返回，对话仍在；清空垃圾桶后本地记录清除）。
