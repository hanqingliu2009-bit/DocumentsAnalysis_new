# 改动日志（验证后登记）

本文件用于记录在**你本地或联调验证通过**后、值得留档的变更：**日期**、**改动原因**、**具体改动**写清楚，便于以后回顾与交接。

- **如何追加**：你验证某次改动 OK 后，在对话里说明「把 xxx 记进改动日志」，维护者按你的描述在下方 **「记录」** 区**倒序追加**（最新在上），或你自行编辑本文件。
- **日期**：以**验证通过当日**为准，格式 `YYYY-MM-DD`；若跨日合并发布，可写区间或主验证日。
- **原因**：业务背景、要解决的问题、关联现象（一两段即可）。
- **改动内容**：涉及的路径、接口、配置项、行为变化，可用列表；必要时写「破坏性变更」「需清数据」等提示。
