# 项目改动总结

本文档汇总截至编写时，在本仓库上做过的一批**功能性、配置与测试**相关改动（含后端、前端、脚本与文档）。便于回顾与交接；**不包含**你本地未提交的临时文件或未跟踪改动，若需与磁盘完全一致请以 `git status` / `git diff` 为准。

---

## 1. 后端（`backend/`）

### 1.1 配置与 LLM（火山引擎）

| 文件 | 改动要点 |
|------|----------|
| `config.py` | 大模型相关环境变量由 `KIMI_*` 更名为 **`VOLCENGINE_API_KEY`**、**`VOLCENGINE_BASE_URL`**；默认 `VOLCENGINE_BASE_URL` 指向火山方舟 OpenAI 兼容地址；**`LLM_MODEL`** 默认空字符串，由 `.env` 填写接入点 ID（如 `ep-...`）。 |
| `.env` | 使用 **`VOLCENGINE_*`** + **`LLM_MODEL`**；具体 Key/URL/接入点以你本地为准（勿提交到 git）。 |

### 1.2 RAG 与聊天（`core/rag.py`、`api/query.py`）

- **`RAGPipeline.query`** 增加 **`retrieval_query`**：聊天场景下用**当前用户一句**做向量检索 embedding，避免把整段对话拼进 `full_question` 导致检索语义被冲淡。
- **`/api/chat`** 调用时传入 **`retrieval_query=request.message.strip()`**。
- 检索到片段但 **未配置 API Key**、或 **未配置 `LLM_MODEL`** 时，返回**明确中文说明**，减少直接抛出长 JSON 错误体。
- **`api/query.py`** 中过时的「Claude」表述改为**通用「已配置的 LLM」**。

### 1.3 文档处理（`core/processor.py`）

- **`_create_chunks`**：在 chunk 已覆盖到文末时 **`break`**，避免短文本场景下循环不前进（曾导致 `test_chunk_short_text` 类用例卡住）。

### 1.4 测试与 pytest（`tests/`、`pytest.ini`）

| 区域 | 改动要点 |
|------|----------|
| `tests/conftest.py` | **`TestClient` 使用 `with TestClient(app) as client:`**，确保 ASGI **lifespan** 执行，`main.document_store` 等不为 `None`；Mock **`EmbeddingGenerator`** 避免测试里真实拉 embedding 模型；`test_settings` 使用 **`VOLCENGINE_*`**；`mock_kimi_client` 重命名为 **`mock_volcengine_client`**。 |
| `tests/api/test_documents.py` | 对 **`main.document_store`** 等的 patch 与下载测试：真实写入 **`FILE_STORAGE_PATH`**，并对 **`api.documents.settings`** patch，避免路由仍用导入时绑定的旧 `settings`。空文件名上传期望 **422**（框架校验先于业务 400）。 |
| `tests/api/test_query.py` | 以 **`get_rag_pipeline`** 等方式 mock，与当前路由实现一致（历史问题已修）。 |
| `tests/core/test_processor.py` | 补充 **`DocumentMetadata`** 导入；**`_extract_text`** 仅传 `file_content`（与实现一致）。 |
| `tests/storage/test_vector_store.py` | fixture teardown 中调用 **`store.client.close()`**，缓解 Windows 下删除临时 Chroma 目录时的 **文件占用** 问题。 |
| `tests/test_config.py` | 与 **`VOLCENGINE_*`**、`LLM_MODEL` 默认值对齐；**`test_default_values`** 显式传入 LLM 相关字段，避免被本机 **`.env`** 污染断言。 |
| `pytest.ini` | 已含 **`pythonpath = .`** 等（若你本地还有调整，以文件为准）。 |

## 2. 前端（`frontend/`）

| 文件 | 改动要点 |
|------|----------|
| `src/services/__tests__/api.test.ts` | **`vi.mock('axios')`** 改为 **`vi.hoisted` + `create` 返回 mock 客户端**，避免 `axios.create` 为 undefined 导致默认导出 `api` 为 undefined；上传/聊天 **`post`** 的第三参数与当前 **`api.ts`**（`timeout`、无手写 multipart `Content-Type`）一致。 |
| `src/pages/ChatInterface.tsx` | 使用 **`axios.isAxiosError`** 解析 **`response.data.detail`**，便于展示后端返回的具体错误；超时仍单独提示。 |
| `package.json` | 增加 **`@vitest/coverage-v8`**，使 **`npm run test:coverage`** 可正常运行。 |
| `package-lock.json` | 随上述依赖安装更新（若存在）。 |

## 3. 根目录脚本与文档

| 文件 | 改动要点 |
|------|----------|
| `start_application.sh` | 使用 **`SCRIPT_DIR`** 定位项目根，去掉他人机器上的硬编码路径；使用 **`backend/venv/bin/python`**；环境变量说明改为 **`VOLCENGINE_*`** / **`LLM_MODEL`**。 |
| `start_servers.sh` | 同上：脚本目录为根、**`backend/venv`** 启动后端。 |
| `README.md` | 架构与前置说明改为 **火山方舟 / Volcengine**；**`.env` 示例**改为 **`VOLCENGINE_API_KEY`**、**`VOLCENGINE_BASE_URL`**、**`LLM_MODEL`**；配置表与栈描述同步。 |

## 4. 行为与排障备忘（与改动相关）

- **Chat 里「打招呼」有时像正常回复**：若向量库未命中片段，会走 **`_no_context_answer`** 的本地模板，**不一定调用大模型**。
- **火山方舟 404 `InvalidEndpointOrModel`**：多为 **`LLM_MODEL`** 与控制台接入点/`model` 不一致；需与 **`VOLCENGINE_BASE_URL`** 所用产品线一致。
- **数据落盘位置**：无独立「业务数据库」；向量在 **`backend/data/vector_db/`**（含 `chroma.sqlite3`），文档元数据在 **`backend/data/documents/*.json`**，上传文件在 **`backend/data/files/`**。
