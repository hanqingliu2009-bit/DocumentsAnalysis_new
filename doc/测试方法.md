# 手动测试清单

在支持 Markdown 的编辑器（或 GitHub / GitLab）中，将 `- [ ]` 改为 `- [x]` 即可标记为已完成。

---

## Chat（前端 Chat 页 → `POST /api/chat`）

- [ ] **无命中 / 与文档无关**：应得到火山返回的**通用回答**，响应里 **`sources` 为空**（或符合当前无检索分支的预期）。
- [ ] **与文档相关且能命中**：仍应是**带引用片段的 RAG**（`sources` 非空，回答依据文档）。

## `POST /api/query`（Swagger `/docs` 或 curl）

- [ ] **无命中**：`sources` 为空、`confidence` / `context_used` 为 0，**`answer` 为固定中文说明**；不应出现闲聊式长回答。

## 配置开关（改 `backend/.env` 后需重启后端）

- [ ] **`CHAT_ALLOW_GENERAL_WITHOUT_DOCS=false`**：Chat 在无命中时应回到**原来的本地说明**，且不调用火山（不消耗 LLM token）。
